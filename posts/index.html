C++ Module

通过cmake来构建C++ Module的项目 编译环境 cmake版本=4.0.3 clang或gcc或者vs2022(注意下支持c++23的版本) ninga 使用STD Module 现有目录如下 项目根目录/ │ ├── CMakeLists.txt │ └── main.cpp // main.cpp import std; int main() { std::println("Hello, C++23 STD Module!"); } 要让其编译成功，需要设置CMakeLists.txt的变量： 设置CXX_MODULE_STD或CMAKE_CXX_MODULE_STD变量为ON。 C++版本设为23。 设置STD Module的实验性UUID。 cmake_minimum_required(VERSION 4.0.3) set(CMAKE_EXPERIMENTAL_CXX_IMPORT_STD "d0edc3af-4c50-42ea-a356-e2862fe7a444") project(hello) # 使用全局变量 set(CMAKE_CXX_MODULE_STD ON) set(CMAKE_CXX_STANDARD 23) add_executable(hello main.cpp) 关于cmake模块支持的文档，可以查看 关于cmake实验性UUID 查询实验性UUID需前往CMake的Github仓库或Gitlab仓库，在项目的Help/dev/experimental.rst文件中，可以找到C++ import std support栏目，提供了相关内容。 需要注意的是，此UUID与CMake版本挂钩，必须使用和你版本匹配的Tag下的UUID。你可以在本地通过cmake --version命令查看本地CMake版本。 最后，生成配置时指定生成器为ninga。 cmake -B build -G "Ninja"
